search-ui.nvim

Zed / IntelliJ の “Find in Files” 級のプロジェクト検索・一括置換を、 Neovim ネイティブの一体型パネルで。構造化フォームで条件を組み、ライブで結果を見て、 diff を確認してから安全に置換する。

土台 自作パネル + fzf-lua 補助 実行 ライブ (ripgrep --json) 置換 diff プレビュー → 一括適用 配色 Selenized 準拠

02

操作フロー

起動から置換適用まで。検索はキー入力のたびにライブ更新され、置換は必ず diff を挟んでから適用します。

1

開く

<leader>sf でパネル起動。前回の条件を引き継がず素の状態から。

→
2

条件を組む

Search / Include / Exclude を入力し、regex・case・word をトグル。Scope を選ぶ。

→
3

ライブで見る

入力停止 120ms で ripgrep 実行。ファイル単位でグルーピングされ即描画。

→
4

絞る / 飛ぶ

CR で該当箇所へジャンプ、Tab で折りたたみ、C-q で quickfix へ。

→
5

置換する

Replace を入れ M-CR。diff を確認しファイルを選び CR で一括適用。

03

機能

.*

正規表現 / リテラル切替

rust regex ベースの正規表現と固定文字列（-F）をワンキーで切替。キャプチャ参照 $1 は置換で利用可。

Aa W

case / word

大文字小文字の区別と単語単位マッチ。未指定時は smart-case。すべてトグルで即反映。

{}

Include / Exclude glob

対象と除外を glob で指定（-g / -g '!…'）。カンマ区切りで複数、fzf-lua でパス候補も選べる。

◎

検索スコープ

Project（git ルート）/ Cwd / 開いているバッファ / 任意パスを切替。任意パスは fzf-lua の picker で。

≈

ライブ検索

入力のたびにデバウンス付きで ripgrep を再実行。進行中の検索はキャンセルし、件数上限で暴走を防止。

±

diff プレビュー置換

実ファイルを触らず置換後を算出し diff 表示。ファイル単位でチェックし、1 undo ブロックで安全に適用。

⚑

quickfix 連携

結果を C-q で quickfix へ。既存の :cdo や他プラグインのワークフローにそのまま橋渡し。

☾

Selenized 準拠テーマ

あなたの selenized.nvim に馴染む配色。light / dark 両対応で環境から浮かない。

05

実装アプローチ（要判断）

「fzf-lua 土台」と「一体型パネル + diff 置換」は技術的に一部両立しません。fzf は単一プロンプトの選択 UI で、複数入力欄フォームや行単位 diff の描画面には使えないためです。落とし所を選んでください。

OPTION A · 推奨

自作パネル + fzf 補助

検索・結果・置換パネルは自作、fzf-lua はスコープ / glob のパス選択 picker として統合。上のモックがこれ。

望む UI を全て満たす · 実装量 中〜大

OPTION B

fzf-lua 全面依存

検索は fzf-lua の live_grep、置換は quickfix → cdo。一体型パネルと diff プレビューは断念。

実装 最小 · 今回の希望とは不一致

OPTION C

grug-far.nvim 採用

望む UI にほぼ一致する既存プラグインを導入し設定で対応。実装ほぼゼロ。

作らない選択肢 ·「自作したい」意図とは別方向

この preview は OPTION A を前提に描いています。方向性が確定したら、M0 スケルトン生成 → M1（ライブ検索）から実装に着手します。詳細な設計は DESIGN.md にあります。
